# MyChordpro 同期方針 概要設計書

## 1. 文書の目的

本書は、`MyChordpro` を `chordwiki_personal` の UX/機能構成へ段階的に寄せるための概要設計を示す。  
ただし、以下は本同期の絶対条件として固定する。

- セットリスト共有機能は **完全にオミット** する
- 認証方式は統合しない（`MyChordpro` は GitHub 認証を維持）
- `ownerId` モデルは統合しない（`MyChordpro` の owner 分離を維持）
- URL 共有で他者へ同一コンテンツを見せる機能は実装しない

## 2. 目標と非目標

### 2.1 目標

- 曲一覧、曲詳細、編集画面の情報設計と操作感を `chordwiki_personal` に近づける
- オートスクロール、表示設定、注釈、YouTube 補助の UX を同等レベルに揃える
- API/データモデルを過不足なく整理し、保守容易性を高める
- 既存ユーザーのデータ分離（ownerId）と認証方式を壊さずに改善する

### 2.2 非目標

- セットリスト機能（作成、共有、URL配布、共同利用）
- Azure AD / Microsoft アカウント連携
- `MyChordpro` と `chordwiki_personal` の ID 空間統合
- マルチユーザーの可視化機能（他人の曲の閲覧、公開リンク）

## 3. 前提・制約

- `MyChordpro` は GitHub 認証を利用する
- `ownerId`（GitHub principal 起点）を partition key とするデータ分離を維持する
- `/api/*` へのアクセスは認証済みユーザーに限定する
- 共有URLを渡しても、アクセス先は受信者自身の認証コンテキストで解決される
- localStorage に保存している個人設定/注釈は既存キー互換を優先する

## 4. 同期方針（アーキテクチャ方針）

### 4.1 方針A: UI/UX 同期

- 画面構成は `トップ / 曲詳細 / 編集` の3軸を維持しつつ操作性を寄せる
- 曲詳細の `Song Controls` と `Tags/YouTube` の配置、折りたたみ、状態保持を継続強化
- オートスクロールのマーカー体験、コンパクト表示、再開挙動を安定化

### 4.2 方針B: API 契約同期（機能限定）

- 曲系 API（一覧、ランキング、検索、詳細、閲覧更新、編集）は現行を主軸に整理
- セットリスト API は追加しない
- 推定 API は `MyChordpro` の運用に必要な範囲に限定し、統合対象外を明示する

### 4.3 方針C: データモデル同期（境界固定）

- Song ドキュメントの必須項目は既存互換で維持
- `ownerId` を中心にしたテナント分離を変更しない
- 表示体験向上に必要な軽微なフィールド追加は許容するが、共有用フィールドは持たない

### 4.4 方針D: セキュリティ境界固定

- 認証は GitHub のまま運用する
- `chordwiki_personal` の MS 認証と統合しない
- UI の editor 表示は `MyChordpro` の認証コンテキストで判定する

## 5. To-Be 機能スコープ

### 5.1 対象（実施する）

- 曲一覧UXの改善（検索導線、ページング、タグ候補、エラー耐性）
- 曲詳細UXの改善（補助パネル、折りたたみ、レスポンシブ）
- 編集導線の改善（曲ページからのタグ/YouTube簡易編集の品質向上）
- 表示/注釈/オートスクロールの状態管理整理

### 5.2 除外（実施しない）

- セットリスト作成・共有・公開
- URL 共有による同一データ閲覧
- 他ユーザーの楽曲探索機能

## 6. データ共有に関する仕様宣言

`MyChordpro` は私的利用専用であり、URL はコンテンツ共有手段にならない。  
ユーザーAが URL をユーザーBへ渡した場合でも、Bには B の認証コンテキスト（ownerId=B）のデータのみが表示される。  
この仕様は本同期で変更しない。

## 7. リスクと対策

- UXのみ寄せる過程で API 契約の差異が混在し、実装が複雑化するリスク  
  - 対策: 「採用する契約」と「採用しない契約」を基本設計書で固定する
- 画面挙動同期で localStorage キー互換を壊すリスク  
  - 対策: キー移行表と後方互換処理を用意する
- ownerId 境界の不整合リスク  
  - 対策: すべての取得/更新 API で ownerId 条件を明示的に検証する
